Project "Budget"

  • Create an application that can keep a record of income and expenses
  • Revenue and expenses are entered by the consumer
  • Class The program has a main method. This main method creates a Budget object
  • In the Budget class do a method called balance () that returns a double / float type number:
    • You get the numbers by summing up all expenses and total income
    • Balance: income minus expenses
    • For example, if your listings are $ 100, $ 20, $ 500, and your costs are $ 200, $ 370, $ 90, the method balance () will return -40.00
  • Add a menu choice so that the user can check the balance at any time
  • It must be possible to print out all income and expenditure at the request of the consumer. Use String.format for formatting
  • Allow the user to select and remove any revenue and expense entry in the lists
  • In the Budget class, there must be a common method for adding an entry
  • In the Budget class, there must be separate methods for generating revenue and expense records
  • Turi būti galimybė regaduoti pasirinktą įrašą:
    • The list of entries is displayed to the user
    • User selects which record to edit (enters record ID)
    • The user is prompted by the values of the current entry and asked if they want to edit this value
  • The class of the budget must have a method of update records
  • The budget program needs to be validated. When the user enters incorrect data (eg the program asks for the amount but the user enters letters) then the program should not break but print the error message and continue (ask again for the amount or return to the main menu). Similarly, all input data must be validated so that the program does not break.
  • Create a Class File with two methods:
    • Save Data — which retrieves a list of records via parameters and saves them to a file in CSV format (values separated by commas).
    • retrieve Data — Which retrieves data from a file, creates record objects, and returns a list of records
  • When a user in the program chooses to save data to a file, then all the records are retrieved from the budget and a method is called to save the data by transferring the list of records

Advanced level project

Egzaminų vertinimo sistema

  • Sistemos tikslas – ištaisyti gautus studentų egzaminus ir įrašyti gautus rezultatus į egzaminų rezultatų failą.
  • Studentai laiko egzaminus internetiniame puslapyje. Atsakius į klausimus, studentas pateikia savo atsakymus į sistemą. Sistema sugeneruoja failus, su kiekvieno studento atsakymais, json formatu.
  • Po kiekvieno egzamino sistema sudeda visus atsakymus į egzamino katalogą. Mums reikalinga programėlė, kuri ištaisytų studentų egzaminus pagal nurodytą teisingų atsakymų failą. Programėle įvertina kiekvieno studento atsakymus, naudodamasi teisingų atsakymaų failu. Gauti studentų įvertinimai surašomi į egzaminų rezultatų failą.
  • Funkciniai reikalavimai:
    • Programa vykdoma iš komandinės eilutės
    • Programa priima du argumentus – kelią iki egzamino katalogo ir kelią iki atsakymų failo. Pavyzdžiui:
      • java exams C:\exams\oop-basics
      • C:\exams\answers\oop_basics_answer.json
    • Rezultatų failas yra sukuriamas jei jis neegzistuoja
    • Teisingai įvykdžius programą rezultatų failas papildomas naujais studentų rezultatais
    • Jau įrašyto egzamino rezultatai nėra rašomi į failą antrą kartą
    • Įvykus klaidai, matomos aiškios klaidos žinutės

    • Examination Evaluation System

  • The purpose of the system is to correct the received student exams and to record the obtained results in the exam result file.
  • Students take exams online. After answering the questions, the student gives their answers to the system. The system generates files, with the answers of each student, in json format.
  • After each exam, the system puts all the answers in the exam directory. We need a gadget to correct student exams according to the correct answer file. The app evaluates each student's answers using the correct answers file. The student grades received are written into the exam result file.
  • Functional requirements:
    • The program is executed from the command line
    • The program accepts two arguments, the path to the exam directory and the path to the answer file. For example:
      • java exams C:\exams\oop-basics
      • C:\exams\answers\oop_basics_answer.json
    • The result file is created if it does not exist
    • Correctly completing the program adds new student results to the results file
    • Already recorded test results are not written to the file a second time
    • When an error occurs, clear error messages are displayed
